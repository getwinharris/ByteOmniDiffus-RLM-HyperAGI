## Current Project Direction

Status: scratch rebuild.

The old byte-search / CoDAR / RLCoDAR prototype is no longer the active product direction.

The active direction is ByteOmniDiffus as a diffusion-style research system around combined remote model-weight records.

A model record groups related remote artifacts as one dense model object:

- GGUF / ONNX / safetensors weights
- Q8 quantization metadata
- config files
- tokenizer files
- model cards
- dataset links
- source URLs
- capability tags

The system is not a next-token prediction pipeline and not a plain byte cosine-search runtime.

## Core Rules

## Project Map Contract

`project_map.mmd` must be generated by `tools/generate_project_map.py`.

The generator must extract repository truth, including:

- folders and files
- AGENTS hierarchy
- schemas
- functions
- tool files
- tests
- configs
- URL/indexing code
- search code
- model artifact handling
- missing expected layers
- stale prototype or legacy terminology

The generated map must make missing systems visible.

Required checks:

```bash
python3 tools/generate_project_map.py
python3 tools/generate_project_map.py --check
pytest -q
```
